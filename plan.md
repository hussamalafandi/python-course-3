# **Python Programming & Data Analysis – Unit-Based Syllabus**

**Target audience:**
Learners with little to no programming experience, progressing to intermediate Python and applied data analysis.

**Overall learning goals:**

* Understand core programming concepts using Python
* Write clean, modular, and testable code
* Apply object-oriented and algorithmic thinking
* Perform real-world data analysis and visualization
* Use professional development workflows (Git, testing, agile methods)

---

## **Unit 1 – Introduction to Programming & Python Ecosystem**

**Purpose:** Establish mental models and tooling before coding in depth.

**Topics:**

* What programming is (problems → algorithms → code)
* How Python works (interpreter, scripts, notebooks)
* Python use cases (automation, data analysis, web, ML)
* Development environments:

  * Jupyter Notebooks
  * VS Code
* Virtual environments:

  * `venv`
  * `conda`
* Installing and managing packages (`pip`, `requirements.txt`)

**Practice:**

* Python installation check
* First Python commands
* Running scripts vs. notebooks

---

## **Unit 2 – Python Fundamentals: Syntax, Variables & Types**

**Purpose:** Build the core language foundation.

**Topics:**

* Python syntax rules and indentation
* Variables and naming conventions
* Built-in data types:

  * `int`, `float`, `str`, `bool`
* Type conversion
* Operators:

  * Arithmetic
  * Comparison
  * Logical
* Basic input/output (`input`, `print`)
* Common beginner errors and debugging basics

**Practice:**

* Simple calculators
* String manipulation exercises
* User input validation (basic)

---

## **Unit 3 – Control Flow & Program Logic**

**Purpose:** Enable decision-making and repetition in programs.

**Topics:**

* Conditional logic:

  * `if`, `elif`, `else`
* Loops:

  * `for`
  * `while`
* Loop control:

  * `break`
  * `continue`
* Boolean logic in practice
* Introductory algorithmic thinking

**Practice:**

* Number guessing game
* Menu-driven programs
* Counting, summing, filtering values

---

## **Unit 4 – Core Data Structures**

**Purpose:** Teach structured data handling before external libraries.

**Topics:**

* Lists
* Tuples
* Dictionaries
* Sets
* Indexing and slicing
* Iterating over collections
* Choosing the right data structure

**Practice:**

* Contact or inventory management
* Word frequency analysis
* Simple data aggregation without Pandas

---

## **Unit 5 – Functions, Modularity & Code Organization**

**Purpose:** Transition from scripts to reusable code.

**Topics:**

* Defining and calling functions
* Parameters and return values
* Default arguments
* Scope (local vs. global)
* Code reuse and decomposition
* Introduction to modules and imports

**Practice:**

* Utility functions library
* Refactoring earlier scripts into functions
* Multi-file Python programs (basic)

---

## **Unit 6 – Mini Project I: Programming Foundations**

**Purpose:** Consolidate foundational skills.

**Example project types:**

* Budget or expense tracker
* Quiz or game logic
* Text-based data processing tool

**Focus areas:**

* Clean structure
* Logical flow
* Readability and correctness

---

## **Unit 7 – Object-Oriented Programming (OOP): Foundations**

**Purpose:** Introduce structured program design.

**Topics:**

* Classes and objects
* Attributes and methods
* Constructors (`__init__`)
* Object state and behavior
* Basic inheritance

**Practice:**

* Domain modeling:

  * Person / Account
  * Product / Order
* Simulating real-world systems

---

## **Unit 8 – Advanced OOP & Design Principles**

**Purpose:** Improve maintainability and scalability.

**Topics:**

* Encapsulation and access control
* Private attributes and properties
* Abstract base classes (`abc`)
* Class methods and static methods
* Composition vs. inheritance
* Design patterns:

  * Factory
  * Singleton

**Practice:**

* Library or employee management system
* Applying Factory Pattern for object creation

---

## **Unit 9 – Computational Thinking & Algorithms**

**Purpose:** Develop problem-solving depth and performance awareness.

**Topics:**

* Computational thinking concepts:

  * Decomposition
  * Pattern recognition
  * Abstraction
* Searching algorithms:

  * Linear search
  * Binary search
* Sorting algorithms:

  * Bubble sort
  * Quick sort
* Algorithm efficiency
* Big-O notation
* Recursion fundamentals

**Practice:**

* Algorithm implementation and comparison
* Recursive problem solving
* Performance analysis exercises

---

## **Unit 10 – Numerical Computing with NumPy**

**Purpose:** Prepare learners for data analysis workflows.

**Topics:**

* NumPy arrays
* Vectorized operations
* Indexing and slicing
* Broadcasting
* Basic numerical statistics

**Practice:**

* Numerical simulations
* Replacing loops with vectorized logic

---

## **Unit 11 – Data Analysis with Pandas**

**Purpose:** Work with real-world datasets.

**Topics:**

* Pandas Series and DataFrames
* Data import:

  * CSV
  * Excel
  * JSON
* Inspecting data
* Handling missing values
* Filtering and sorting
* Grouping and aggregation

**Practice:**

* Cleaning messy datasets
* Exploratory data analysis tasks

---

## **Unit 12 – Data Visualization & Reporting**

**Purpose:** Communicate insights effectively.

**Topics:**

* Visualization fundamentals
* Matplotlib:

  * Line, bar, scatter plots
* Seaborn overview
* Choosing the right chart
* Basic storytelling with data

**Practice:**

* Visual EDA notebooks
* Mini analytical reports

---

## **Unit 13 – Version Control, Prototyping & Agile Methods**

**Purpose:** Introduce professional development workflows.

**Topics:**

* Git fundamentals:

  * init, add, commit
  * branches
* GitHub collaboration basics
* Prototyping concepts
* Agile overview:

  * Scrum roles and ceremonies
* Design Thinking (empathize → prototype)

**Practice:**

* Versioning a Python project
* Collaborative workflow simulation

---

## **Unit 14 – Testing, Quality Assurance & Clean Code**

**Purpose:** Emphasize reliability and maintainability.

**Topics:**

* Why testing matters
* Unit testing basics
* `unittest` / `pytest`
* Writing testable code
* Code reviews
* Clean Code principles

**Practice:**

* Writing tests for existing code
* Refactoring with tests in place

---

## **Unit 15 – Capstone Project, Review & Assessment**

**Purpose:** Demonstrate integrated competence.

**Components:**

* End-to-end project:

  * Software-focused **or**
  * Data analysis–focused
* Version control required
* Tests where applicable
* Final presentation or report

**Assessment criteria:**

* Correctness and robustness
* Code quality and structure
* Problem-solving approach
* Use of tools and best practices
